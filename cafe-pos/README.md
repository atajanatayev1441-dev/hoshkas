# ☕ ЛаунжКасса — POS-система для кафе

Кассовая система с PostgreSQL, ESC/POS чеками и бухгалтерской аналитикой.

## Страницы
- `/` — **Касса**: приём заказов, корзина, печать чека
- `/accounting` — **Бухгалтерия**: выручка, заказы, топ позиций, экспорт CSV
- `/menu` — **Меню**: управление категориями и блюдами

---

## Локальный запуск

### 1. Требования
- Node.js 18+
- PostgreSQL (локально или облако)

### 2. Сервер
```bash
cd server
npm install
```

Создайте файл `.env`:
```
DATABASE_URL="postgresql://user:password@localhost:5432/cafe_pos"
```

```bash
npx prisma db push       # создать таблицы
node seed.js             # заполнить меню
npm run dev              # запустить сервер на :3001
```

### 3. Клиент
```bash
cd client
npm install
npm run dev              # запустить на :5173
```

Открыть: http://localhost:5173

## Деплой на Railway

1. Создайте новый проект на [railway.app](https://railway.app)
2. Подключите GitHub репозиторий
3. Добавьте сервис **PostgreSQL** из каталога Railway
4. Railway автоматически добавит `DATABASE_URL` в переменные окружения
5. Деплой запустится автоматически (railway.toml уже настроен)
6. После первого деплоя выполните сид:
   ```
   railway run node server/seed.js
   ```

## Печать чеков (ESC/POS)

### Способ 1 — Web Serial API (рекомендуется)
- Браузер: **Google Chrome** или Edge
- Принтер подключён по USB или COM-порту
- При первом заказе Chrome попросит выбрать порт принтера
- Работает без драйверов, напрямую через браузер

### Способ 2 — Авто-fallback
Если Web Serial недоступен, автоматически откроется окно браузерной печати (подходит для любого принтера через Windows/Mac)

### Совместимые принтеры
Любой ESC/POS термопринтер: АТОЛ, Epson TM, Star, Citizen, Xprinter и др.

## Структура базы данных

```
Category  →  Item  →  OrderItem  →  Order
```

- **Category**: категории меню
- **Item**: позиции меню (название, цена, категория)
- **Order**: заказы (номер, стол, итого, тип оплаты, статус)
- **OrderItem**: позиции в заказе (снимок цены и названия на момент продажи)

## API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/categories | Все категории с товарами |
| POST | /api/categories | Создать категорию |
| GET | /api/items | Все позиции |
| POST | /api/items | Добавить позицию |
| PUT | /api/items/:id | Редактировать позицию |
| POST | /api/orders | Создать заказ |
| GET | /api/orders | Список заказов (фильтры: status, from, to) |
| GET | /api/analytics/summary | Аналитика за период |
| GET | /api/export/orders | Выгрузить CSV |
